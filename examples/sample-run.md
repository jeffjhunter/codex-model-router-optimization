# Illustrative run

> This is a design example, not an authenticated Codex transcript or benchmark result.

## Request

```text
$route-codex-work Add a --json flag to the local status command. Preserve text output, add focused tests, and keep all work local.
```

## 1. Root contract

The Sol coordinator records numbered requirements for JSON output, backward compatibility, tests, and local-only scope. It creates atomic criteria, captures the Git baseline, scores the task for Terra, resolves the exact saved local project, and records the app-task backend and actor-contract hash.

See [plan-packet.yaml](plan-packet.yaml).

## 2. Worker identity and attempt 1

The coordinator creates a Terra/high task with the no-write preflight contract. It waits until that task is idle, captures the completed preflight turn ID, and binds independent session evidence to the exact task, turn, model, effort, and repository CWD.

Only then does the root send the Terra actor contract and implementation plan to the retained task. The worker implements the flag and tests, then returns a structured report with commands, exits, changed paths, and limitations. After that turn is idle, the coordinator captures and independently verifies its exact completed turn before accepting the report.

See [worker-report.yaml](worker-report.yaml).

## 3. Independent review

After the writer is idle, a separate Sol/xhigh task passes its own read-only identity preflight. The coordinator takes a content snapshot and then sends the review contract. The reviewer inspects the diff and reruns focused tests. It finds that redirected text output changed its final newline, marks backward compatibility failed, and requests the observable old behavior—not a prescribed code edit. The coordinator verifies the exact completed review turn and requires the post-review snapshot digest to match.

See [review-revise.yaml](review-revise.yaml).

## 4. Same-worker revision

The coordinator sends only the failed criterion, reproduction, and requested outcome to the retained writer task ID as attempt 2, with no model override. The worker fixes the behavior and updates its evidence; the coordinator authenticates that exact revision turn before rereview.

## 5. Review and root gate

The retained reviewer accepts the current plan version. The root confirms requirement coverage, every material task-and-turn identity, current paths, test evidence, matching review snapshot digests, and local-only scope before returning complete.

The key properties are explicit model selection, runtime observation, identity continuity, current-version evidence, independent artifact review, and a root gate. This remains an illustrative trace until reproduced as a sanitized observed run.
